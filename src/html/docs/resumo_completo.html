🎯 1. Visão Geral do Projeto

O que é?

FSS Design Lab é um simulador educacional interativo de Superfícies Seletivas de Frequência (FSS). Permite explorar 4 geometrias diferentes com visualização em tempo real.

Tipo: Aplicação Web Estática (HTML + JavaScript)
Framework: HTML5, CSS3, JavaScript ES6 Modules
Bibliotecas: Chart.js (via CDN) para gráficos
Modelos Suportados: 4 geometrias FSS diferentes
Funcionalidades: Simulação, visualização, análise, exportação CSV

Objetivos Principais

  • Educar sobre FSS através de simulação interativa
  • Visualizar geometrias e resposta em frequência em tempo real
  • Permitir exploração de parâmetros e seus efeitos
  • Oferecer múltiplos modelos para comparação
  • Facilitar análise e exportação de dados

Padrão de Arquitetura: Modular

Cada página HTML corresponde a um modelo diferente, com seu próprio controlador (módulo ES6).

  • Separação de Responsabilidades: cada módulo cuida de uma tarefa específica
  • Reutilização: funções matemáticas e visuais são compartilhadas
  • Escalabilidade: novos modelos podem ser adicionados facilmente

📐 src/js/math.js

Utilitários matemáticos compartilhados: conversões, funções trigonométricas, cálculos de impedância.

📊 src/js/visual.js

Utilitários de visualização: criação de gráficos Chart.js, exportação de dados em CSV.

🎨 src/css/styles.css

Estilos centralizados aplicados a todas as páginas simuladoras. Layout responsivo com CSS Grid.

🔬 3. Os 4 Modelos FSS Implementados

1️⃣ Cruz de Jerusalém (Cruzjeru.html)

Geometria: Padrão em forma de cruz com braços de comprimento d e espessura w

Parâmetros: p, d, w, h, g (5 parâmetros)

Modelo Matemático: Langley-Munk com ajustes para 5-polos

Características:

  • ✓ Multirressonante (múltiplos picos)
  • ✓ Visualização com 5 dimensões anotadas
  • ✓ Ideal para filtros complexos
Langley-Munk Completo

2️⃣ Patch Quadrado (Quadrado.html)

Geometria: Quadrado sólido de lado c repetido em grade p×p

Parâmetros: p, c (2 parâmetros simplificados)

Modelo Matemático: Ulrich (sub-ressonante)

Características:

  • ✓ Modelo mais simples
  • ✓ Filtro passa-baixa
  • ✓ Interface intuitiva
Ulrich Completo

3️⃣ Espira Quadrada (EspiraQuadrada.html)

Geometria: Anel quadrado de fio metálico com diâmetro d e espessura w

Parâmetros: p, d, w, g (4 parâmetros)

Modelo Matemático: Langley com adaptações para loop

Características:

  • ✓ Resposta banda-larga
  • ✓ Acoplamento capacitivo
  • ✓ Visualização com 4 dimensões
Langley-Loop Completo

4️⃣ Anel Circular (anelcircular.html)

Geometria: Anel circular de fio metálico com raio r e espessura w

Parâmetros: p, r, w, g (4 parâmetros)

Modelo Matemático: Equivalência Geométrica

Características:

  • ✓ Simetria circular
  • ✓ Filtro rejeita-banda
  • ✓ Dimensões responsivas
Geométrico Completo

🎨 4. Sistema de Visualização

Canvas 2D - Geometria

Cada página exibe uma área 400×400 px onde a geometria FSS é desenhada em tempo real.

  • Grade 3×3 de células (central + 8 vizinhas)
  • Célula central em azul escuro destaque
  • Vizinhas em azul claro/transparente
  • Linha tracejada mostrando período
  • Dimensões com setas coloridas e anotações

Gráficos - Chart.js

Gráfico de linha mostrando S21 vs Frequência.

  • Eixo X: Frequência em GHz
  • Eixo Y: S21 em dB
  • Pontos principais marcados: fr (frequência de ressonância)
  • Múltiplos datasets para comparação de fórmulas

Sistema de Dimensões Responsivo

As setas, textos e rótulos se ajustam automaticamente ao tamanho do canvas.

fontSize = Math.max(10, pPixel * 0.06)
arrowSize = Math.max(3, pPixel * 0.03)
lineWidth = Math.max(1.5, pPixel * 0.008)
offset = Math.max(25, pPixel * 0.15)

Onde pPixel é o período em pixels no canvas (p × scale).

Legenda Colorida

Cada modelo exibe uma legenda mostrando o significado das cores usadas nas dimensões:

  • 🔴 Vermelho: Período (p)
  • 🟠 Laranja: Largura do fio (w)
  • 🔵 Azul: Gap/espaçamento (g)
  • 🟣 Roxo: Dimensões específicas (h, etc)

▶️ 5. Fluxo de Execução

Quando você abre uma página simuladora:

  1. HTML carrega Chart.js via CDN
  2. HTML carrega CSS centralizado (styles.css)
  3. Módulo ES6 correspondente (ui_*.js) é injetado com type="module"
  4. Função init() executa quando DOM está pronto (DOMContentLoaded)
  5. Controles (sliders, inputs) são ligados aos eventos de mudança
  6. Parâmetros iniciais são aplicados
  7. Simulação é executada pela primeira vez

Quando você muda um parâmetro:

  1. Evento de mudança dispara (input, change ou DOMManipulation)
  2. Valores dos parâmetros são atualizados
  3. Validações são aplicadas (limites min/max)
  4. Função updateAll() é chamada (orquestrador)
  5. Geometria é redesenhada no canvas
  6. Simulação matemática é recalculada (loop em frequência com df=0.001 GHz)
  7. Gráfico é atualizado com novos dados
  8. Frequência de ressonância (fr) e largura de banda (BW) são exibidas

Quando você clica em "Download CSV":

  1. Função exportChartToCSV(chart, filename) é chamada
  2. Dados do gráfico são extraídos em formato tabular
  3. Arquivo CSV é gerado com headers: "Frequency (GHz);S21 (dB)"
  4. Download é iniciado no navegador

📐 6. Técnicas Matemáticas Utilizadas

Modelo de Circuito Equivalente (ECM)

FSS é modelada como um circuito RLC equivalente. A impedância é calculada através de fórmulas especializadas para cada geometria.

Fórmula de Transmissão - S21

Baseada na impedância equivalente Z e na impedância do espaço livre:

S21 = 20 * log10(2 / |1 + Z/Z0|)
onde Z0 = 377 Ω (impedância do espaço livre)

Cálculo da Permissividade Efetiva

Quando um substrato está presente, a permissividade efetiva é calculada usando diferentes fórmulas:

  • Costa 2020 (com α dinâmico): Considera efeito de borda
  • Exponencial: Função exponencial suave
  • Tanh: Função tangente hiperbólica
  • Puro: Apenas o valor nominal (εᵣ)

Cálculo de fr (Frequência de Ressonância)

É a frequência onde S21 é mínimo. Detectada varredurando o array completo de frequências e encontrando o índice do valor mínimo.

Cálculo de BW (Largura de Banda)

Definida como a faixa entre os pontos de -3dB em relação ao máximo (em geral 0 dB). É a largura da "janela" de bloqueio.

🚀 8. Guia para Adicionar Novos Modelos

Passos para Adicionar um Novo Modelo FSS:

1️⃣ Criar arquivo HTML

Exemplo: novo_modelo.html seguindo a estrutura dos existentes:

<!DOCTYPE html>
<html>
<head>...</head>
<body>
  <div class="dashboard">...(painel params e canvas)...</div>
  <script type="module" src="src/js/ui_novo.js"></script>
</body>
</html>

2️⃣ Criar módulo JavaScript

Exemplo: src/js/ui_novo.js com funções essenciais:

import { FF, GG } from './math.js';
import { createLineChart, exportChartToCSV } from './visual.js';

function drawGeometry(p, param1, param2) { ... }
function updateAll() { ... }
function init() { ... }

document.addEventListener('DOMContentLoaded', init);

3️⃣ Implementar funções principais

  • drawGeometry(): desenha a geometria no canvas
  • updateAll(): orquestra atualização de tudo
  • Simulação: loop em frequência com cálculos
  • init(): inicia e liga eventos

4️⃣ Reutilizar funções comuns

  • Cálculos de fórmulas de math.js
  • Criação de gráficos via createLineChart() de visual.js
  • Exportação CSV via exportChartToCSV()
  • Estilos centralizados de styles.css

5️⃣ Adicionar link no index.html

Incluir card com link na dashboard principal.

Boas Práticas

  • Mantenha separação entre lógica (math.js) e apresentação (ui_*.js)
  • Use variáveis descritivas em português
  • Documente com comentários explicando cada seção
  • Valide parâmetros (mín, máx, tipo)
  • Garanta responsividade com cálculos proporcionais ao canvas
  • Teste com múltiplos valores de entrada
