Resumo Completo do Projeto

Documentação técnica completa do FSS Explorer: arquitetura de software, modelos implementados, sistema de visualização e guia para extensões.

1. Visão Geral do Projeto

O que é o FSS Explorer?

FSS Explorer é um simulador educacional interativo de Superfícies Seletivas de Frequência (FSS). Permite explorar 6 geometrias diferentes com visualização em tempo real.

PropriedadeValor
TipoAplicação Web Estática (HTML + JavaScript)
StackHTML5, CSS3, JavaScript ES6 Modules
BibliotecasChart.js (CDN) para gráficos
Modelos6 geometrias FSS diferentes
FuncionalidadesSimulação, visualização, análise, exportação CSV

Objetivos Principais

  • Educar sobre FSS através de simulação interativa
  • Visualizar geometrias e resposta em frequência em tempo real
  • Permitir exploração de parâmetros e seus efeitos
  • Oferecer múltiplos modelos para comparação
  • Facilitar análise e exportação de dados

3. Os 6 Modelos FSS Implementados

Quase Quadrado — quasequadrado.html

Geometria: Topologia Quasi-Square Loop com gaps centrais.

Parâmetros: p, w, g1, g2 (4 parâmetros)  |  Modelo: Mamedes (2024)

  • Calibração capacitiva dinâmica (KL_AUTO)
  • Análise banda-larga 5G
  • Geometria detalhada em SVG/Canvas
Mamedes ECM Completo

Estrela de 4 Pontas — estrela4pontas.html

Geometria: Tapered Star com quadrado central e pontas afiladas.

Parâmetros: p, a, b, s (4 parâmetros)  |  Modelo: Mamedes (2024)

  • Altíssima capacitância de borda compensada
  • Operação mm-wave (ex: 28 GHz)
  • Fidelidade geométrica baseada no artigo
Mamedes ECM Completo

Cruz de Jerusalém — cruzjeru.html

Geometria: Padrão em forma de cruz com braços de comprimento d e espessura w.

Parâmetros: p, d, w, h, g (5 parâmetros)  |  Modelo: Langley-Munk

  • Multirressonante (múltiplos picos)
  • Visualização com 5 dimensões anotadas
  • Ideal para filtros complexos
Langley-Munk Completo

Patch Quadrado — quadrado.html

Geometria: Quadrado sólido de lado c repetido em grade p×p.

Parâmetros: p, c (2 parâmetros)  |  Modelo: Ulrich (sub-ressonante)

  • Filtro passa-baixa
  • Interface suspensa para ajustes teóricos
Ulrich Em Manutenção

Espira Quadrada — espira.html

Geometria: Anel quadrado de fio metálico com diâmetro d e espessura w.

Parâmetros: p, d, w, g (4 parâmetros)  |  Modelo: Langley-Loop

  • Resposta banda-larga
  • Acoplamento capacitivo
  • Visualização com 4 dimensões
Langley-Loop Completo

Anel Circular — anelcircular.html

Geometria: Anel circular de fio metálico com raio r e espessura w.

Parâmetros: p, r, w, g (4 parâmetros)  |  Modelo: Equivalência Geométrica

  • Simetria circular
  • Filtro rejeita-banda
  • Dimensões responsivas
Geométrico Completo

4. Sistema de Visualização

Canvas 2D — Geometria

Cada página exibe uma área 400×400 px onde a geometria FSS é desenhada em tempo real.

  • Grade 3×3 de células (central + 8 vizinhas)
  • Célula central em azul escuro com destaque
  • Linha tracejada indicando o período
  • Dimensões com setas coloridas e anotações

Gráficos — Chart.js

Gráfico de linha mostrando S21 vs Frequência.

  • Eixo X: Frequência em GHz
  • Eixo Y: S21 em dB
  • Ponto marcado: fr (frequência de ressonância)
  • Múltiplos datasets para comparação de fórmulas

Sistema de Dimensões Responsivo

Setas, textos e rótulos se ajustam automaticamente ao tamanho do canvas:

fontSize = Math.max(10, pPixel * 0.06) arrowSize = Math.max(3, pPixel * 0.03) lineWidth = Math.max(1.5, pPixel * 0.008) offset = Math.max(25, pPixel * 0.15)

Onde pPixel é o período em pixels no canvas.

5. Fluxo de Execução

Ao abrir uma página simuladora

  1. HTML carrega Chart.js via CDN e CSS centralizado
  2. Módulo ES6 correspondente (ui_*.js) é injetado com type="module"
  3. Função init() executa quando DOM está pronto (DOMContentLoaded)
  4. Controles (sliders, inputs) são ligados a eventos de mudança
  5. Simulação é executada pela primeira vez

Ao alterar um parâmetro

  1. Evento dispara — validações min/max são aplicadas
  2. Função updateAll() é chamada (orquestrador)
  3. Geometria é redesenhada no canvas
  4. Simulação recalculada (loop em frequência com df = 0.001 GHz)
  5. Gráfico atualizado — fr e BW são exibidos no painel

Ao clicar em "Download CSV"

  1. Função exportChartToCSV(chart, filename) é chamada
  2. Dados do gráfico são extraídos em formato tabular
  3. CSV gerado com headers: "Frequency (GHz);S21 (dB)"
  4. Download iniciado no navegador

6. Técnicas Matemáticas Utilizadas

Modelo de Circuito Equivalente (ECM) e Modelo Físico L & C

FSS é modelada como um circuito RLC equivalente (ECM). As reatâncias resultantes (XL e BC) são convertidas diretamente e exibidas na interface como um Modelo Físico equivalente de Indutância (L) em nH e Capacitância (C) em pF.

Fórmula de Transmissão — S21

Baseada na impedância equivalente Z e na impedância do espaço livre:

S21 = 20 × log₁₀(2 / |1 + Z/Z0|) onde Z0 = 377 Ω (impedância do espaço livre)

Cálculo da Permissividade Efetiva

Quando um substrato está presente, a permissividade efetiva é calculada por diferentes modelos:

  • Costa 2020 (com α dinâmico): Considera efeito de borda
  • Exponencial: Função exponencial suave
  • Tanh: Função tangente hiperbólica
  • Puro: Apenas o valor nominal (εᵣ)

Frequência de Ressonância (fr)

É a frequência onde S21 é mínimo. Detectada varrendo o array completo de frequências.

Largura de Banda (BW)

Faixa entre os pontos de −3 dB em relação ao máximo. É a largura da "janela" de bloqueio.

8. Guia para Adicionar Novos Modelos

Passo 1 — Criar arquivo HTML

Criar src/html/topologies/novo_modelo.html seguindo a estrutura dos existentes.

<!DOCTYPE html> <html> <head>...</head> <body> <div class="dashboard">...(painel params e canvas)...</div> <script type="module" src="../../js/topologies/ui_novo.js"></script> </body> </html>

Passo 2 — Criar módulo JavaScript

Criar src/js/topologies/ui_novo.js com as funções essenciais:

import { FF, GG } from '../math.js'; import { createLineChart, exportChartToCSV } from '../visual.js'; function drawGeometry(p, param1, param2) { ... } function updateAll() { ... } function init() { ... } document.addEventListener('DOMContentLoaded', init);

Passos 3–5 — Implementar, reutilizar, publicar

  • drawGeometry(): desenha a geometria no canvas
  • updateAll(): orquestra atualização de tudo
  • Simulação: loop em frequência com cálculos matemáticos
  • init(): inicia e liga eventos de DOM
  • Reutilize createLineChart() e exportChartToCSV() de visual.js
  • Adicione um card no index.html com o link para a nova topologia
Boas práticas: mantenha separação entre lógica (math.js) e apresentação (ui_*.js). Use variáveis descritivas, documente com comentários, valide parâmetros (mín/máx) e garanta responsividade com cálculos proporcionais ao canvas.
